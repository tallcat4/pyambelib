# pyambelib

`mbelib-neo`のPythonラッパーライブラリです。
ビットインターリーブを実装し、これによりハードウェアボコーダーチップと等価なフォーマットとして扱えるようになっています。

## インストール

```bash
pip install .
```

## 使い方 (Usage)

### `AmbeDecoder` クラス

音声のデコード処理を行うメインのクラスです。複数チャネルを扱う場合でも、インスタンスごとに内部状態が独立して保持されます。
これにより、複数チャネルの同時デコードなどが容易に実装できます。

* `decode_2450(payload)`: 7バイトのデータ (AMBE2450形式) を受け取り、160サンプルのPCM音声にデコードします。
* `decode_3600(payload)`: 9バイトのデータ (AMBE3600x2450形式) を受け取り、同じく160サンプルのPCM音声にデコードします。

### `fec_demod` 関数

AMBE3600x2450 の 9バイトデータから前方誤り訂正 (FEC) を復調し、49ビットのデータ (AMBE2450形式) を抽出します。

> ちなみに、ARIB STD-T98の仕様書には、AMBEフレームのFECの方法が記載されていません。チップのデータシートにも書かれていません。
> そしてどういうわけか、後述の引用にあるProject 25の企画書にそれが記載されています。
> まだ試していませんが、このfec_demodは、STD-T98のスクランブルを解読するための最初の処理として利用できるはずです。


**独自インターリーブ実装について:**
オリジナルである `mbelib`および `mbelib-neo` には、設計意図が（少なくとも私には）不明なビットインターリーブ処理が含まれています。
本ライブラリではそのインターリーブ規則を独自に解釈・再実装しています。
これにより、抽出したデータを ThumbDV などのハードウェアボコーダーチップと等価なフォーマットとして扱えるようになっています。
